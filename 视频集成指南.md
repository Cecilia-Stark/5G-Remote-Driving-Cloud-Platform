## 🚀 使用步骤

### 第 1 步：启动小车程序

在小车端运行：

```bash
# SSH 登录小车
ssh nvidia@小车IP

# 运行控制脚本
cd ~
./start_robot.sh
```

看到以下输出说明成功：

```
==========================================
  小车控制服务器启动成功！
==========================================

  视频流地址：http://192.168.0.5:8000/mjpeg
  控制接口：http://192.168.0.5:8000/control
  G29 监听端口：5000
```

---

### 第 2 步：启动低延迟视频代理（电脑上）

**方式 A: 使用批处理脚本**

```bash
# 双击运行
start-lowlatency.bat
```

**方式 B: 命令行启动**

```bash
# 编辑小车 IP（如果需要）
# 修改 video-proxy-lowlatency.cjs 中的 TARGET_HOST

# 启动代理
node video-proxy-lowlatency.cjs
```

你会看到：

```
============================================================
📺 低延迟视频流代理
============================================================
目标：http://192.168.0.5:8000/mjpeg
本地：http://localhost:8001
============================================================

✅ 代理服务器已启动

  本地访问：http://localhost:8001/
  远程访问：http://你的电脑 IP:8001/
```

---

### 第 3 步：启动 G29 方向盘服务（电脑上）

```bash
# 双击运行
start-all.bat

# 或单独启动 G29 服务
node g29-server.cjs
```

---

### 第 4 步：启动 Web 平台（电脑上）

```bash
# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

浏览器访问：`http://localhost:5173`

---

### 第 5 步：配置小车 IP（Web 界面）

1. 登录 Web 平台
2. 进入**驾驶舱页面**（ControlView）
3. 点击右上角 **"小车 IP"** 按钮
4. 输入小车 IP 地址（如 `192.168.0.5`）
5. 点击 **"保存并重启"**

## 🔧 故障排查

### 问题 1: 视频一直显示"连接中..."

**检查清单**:

```bash
# 1. 小车程序是否运行
ssh nvidia@小车IP "ps aux | grep robot_web_control"

# 2. 测试小车视频流
curl http://小车IP:8000/mjpeg

# 3. 测试代理服务器
curl http://localhost:8001/

# 4. 检查网络连通性
ping 小车IP
```

**解决步骤**:

1. 确认小车程序运行正常
2. 重启视频代理：`node video-proxy-lowlatency.cjs`
3. 检查防火墙是否开放 8000、8001 端口

---

### 问题 2: 视频很卡或延迟仍然很高

**优化方案**:

1. **降低分辨率**（小车端）
   
   编辑小车摄像头配置，降低分辨率：
   ```bash
   # 从 1080p 降到 720p
   1920x1080 → 1280x720
   ```

2. **降低帧率**
   
   ```bash
   # 30fps → 20fps
   framerate=20
   ```

3. **使用 5GHz WiFi**
   
   确保电脑和小车都连接 5GHz 频段

4. **直接访问**（同局域网）
   
   修改 `src/services/carControl.ts`:
   ```typescript
   const VIDEO_URL = `http://${CAR_IP}:8000/mjpeg`;  // 不走代理
   ```

---

### 问题 3: 画面花屏或闪烁

**可能原因**:
- 网络丢包
- 缓冲区溢出
- 编码器问题

**解决方法**:

```bash
# 1. 检查网络质量
ping -c 100 小车IP

# 2. 查看丢包率
# 如果 > 1%，考虑使用 5GHz WiFi 或有线连接

# 3. 降低码率（小车端）
# 编辑 robot_web_control.py 中的 VIDEO_PORT 配置
```

## 🛠️ 进阶配置

### 修改默认视频源

编辑 `src/components/CarVideoFeed.tsx`:

```typescript
// 第 24 行：修改默认视频流地址
const defaultSrc = 'http://localhost:8001/';  // 本地代理
// 或
const defaultSrc = 'http://192.168.0.5:8000/mjpeg';  // 直连小车
```

### 添加多个摄像头

在 `ControlView.tsx` 中添加多个 `CarVideoFeed`:

```typescript
<CarVideoFeed
  label="前视摄像头"
  src="http://localhost:8001/front"
/>
<CarVideoFeed
  label="后视摄像头"
  src="http://localhost:8001/rear"
/>
```

### 启用自动重连

`CarVideoFeed` 已默认启用 3 秒自动重连，无需额外配置。
