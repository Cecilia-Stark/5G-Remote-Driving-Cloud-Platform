## 📋 配置步骤

### 第 1 步：在小车上查看 IP 地址

在小车终端运行：
```bash
hostname -I
```

记录输出的 IP 地址，例如：`192.168.1.100`

---

### 第 2 步：在小车上运行控制脚本

在小车终端运行你的启动脚本：
```bash
./run
# 或者
./start_robot.sh
```

你应该看到类似输出：
```
==========================================
  小车控制服务器启动成功！
==========================================

  视频流地址：http://192.168.1.100:8000/mjpeg
  控制接口：http://192.168.1.100:8000/control
  G29 监听端口：5000
```

---

### 第 3 步：验证小车视频流

在小车浏览器访问：
```
http://localhost:8000/mjpeg
```

或者在电脑浏览器访问：
```
http://小车IP:8000/mjpeg
```

能看到视频画面说明小车端正常！✅

---

### 第 4 步：在 Web 平台配置小车 IP

#### 方法 A：通过 Web 界面配置（推荐）

1. 启动 Web 平台：
   ```bash
   cd E:\OneDrive\桌面\tt\5G-Remote-Driving-Cloud-Platform
   npm run dev
   ```

2. 浏览器访问：`http://localhost:5173`

3. 登录 → 监控页面 → 点击"立即接管"

4. 点击右上角 **"小车 IP"** 按钮

5. 输入小车 IP（如 `192.168.1.100`）

6. 点击 **"保存并重启"**

#### 方法 B：直接修改代码

编辑 `src/services/carControl.ts` 第 5 行：
```typescript
let CAR_IP = localStorage.getItem('car_ip') || '192.168.1.100';
//                                        ↑ 改成你的小车 IP
```

#### 方法 C：浏览器控制台设置

按 `F12` 打开控制台，输入：
```javascript
localStorage.setItem('car_ip', '192.168.1.100');
location.reload();
```

---

### 第 5 步：验证 Web 平台视频

在 Web 平台的驾驶舱页面（ControlView），左侧应该显示小车实时视频。

如果显示"视频流连接中..."，请检查：
1. 小车程序是否运行
2. 小车 IP 配置是否正确
3. 电脑和小车是否在同一网络

## 🔧 端口转发方案（如果直接访问失败）

如果电脑无法直接访问小车 IP，可以使用 SSH 端口转发：

### 在电脑上运行：
```bash
# 通过 SSH 将小车 8000 端口转发到本地
ssh -L 8000:localhost:8000 nvidia@小车IP
```

### 然后在 Web 平台配置：
1. 打开 Web 界面配置
2. 输入小车 IP 为：`localhost`
3. 保存并重启

这样 Web 平台会通过 `http://localhost:8000/mjpeg` 访问视频流。

## ⚠️ 常见问题

### 问题 1：视频一直显示"连接中..."
**解决：**
```bash
# 在小车测试本地视频
curl http://localhost:8000/mjpeg

# 检查小车防火墙
sudo ufw allow 8000/tcp
sudo ufw allow 5000/tcp
```

### 问题 2：电脑无法访问小车
**解决：**
```bash
# 在电脑 ping 小车
ping 小车IP

# 如果不通，检查是否在同一 WiFi
# 或使用 SSH 端口转发
```

### 问题 3：Web 平台配置不保存
**解决：** 浏览器清除了 localStorage，每次需要重新配置，或使用代码修改默认 IP。
