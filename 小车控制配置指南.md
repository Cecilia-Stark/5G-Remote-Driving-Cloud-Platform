# 🚗 小车视频传输与控制配置指南

## ✅ 已完成功能

### 1. 小车视频流显示
- 使用 MJPEG 协议实时显示小车摄像头画面
- 自动显示 FPS 帧率和连接状态
- 支持视频流错误检测和重连

### 2. G29 方向盘控制小车
- 读取 G29 方向盘、油门、刹车数据
- 每 50ms 发送一次控制指令到小车
- 踩刹车时自动停止

## 📋 使用步骤

### 第一步：配置小车 IP 地址

1. **在小车上查看 IP 地址**
   ```bash
   # 在小车终端运行
   ifconfig
   # 或
   ip addr show
   ```
   找到类似 `192.168.1.100` 的 IP 地址

2. **在 Web 界面配置**
   - 启动 Web 项目：`npm run dev`
   - 登录后进入监控页面
   - 点击 "立即接管" 进入驾驶舱
   - 点击右上角 "小车 IP" 按钮
   - 输入小车 IP 地址，保存并重启

### 第二步：确保小车程序运行

在小车上运行你的 ROS2 控制程序：

```bash
cd /path/to/your/robot-code
python3 robot_control.py
```

你应该看到类似输出：
```
控制页面：http://192.168.1.100:8000
G29 监听端口：6999
```

### 第三步：启动 G29 服务

在你的电脑上运行 G29 Node.js 服务：

```bash
node g29-server.js
```

### 第四步：开始控制

1. 在 Web 界面点击 "立即接管"
2. 确认 G29 方向盘已连接（看到绿色状态）
3. 确认小车视频已显示（看到绿色信号点和 FPS）
4. 踩油门踏板，小车应该开始移动
5. 转动方向盘，小车应该转向
6. 踩刹车，小车应该停止

## 🔧 故障排查

### 问题 1: 视频显示 "连接中..."
**原因：** 小车 IP 配置错误或小车程序未运行

**解决：**
1. 检查小车 IP 是否正确
2. 在小车浏览器访问 `http://localhost:8000/mjpeg` 测试
3. 确保小车程序正在运行

### 问题 2: G29 显示 "设备未就绪"
**原因：** Node.js G29 服务未运行

**解决：**
```bash
# 检查服务是否运行
node g29-server.js
```

### 问题 3: 控制指令发送失败
**原因：** 网络不通或小车程序未响应

**解决：**
1. 检查电脑和小车是否在同一局域网
2. 在小车终端 ping 你的电脑
3. 检查防火墙是否阻止 8000 端口

### 问题 4: 车不对控制产生响应
**原因：** 控制模式可能在键盘模式

**解决：**
- 在小车网页上按 `K` 键切换到 G29 模式
- 或者在代码中设置 `use_keyboard = false`

## 📐 控制参数说明

### 速度映射
```
G29 油门 0-100% → 小车速度 0-2.0 m/s
G29 刹车 >10%   → 小车停止
G29 方向盘 ±45°  → 小车转向 ±1.5 rad/s
```

### 控制频率
- 每 **50ms** 发送一次指令
- 与小车 ROS2 控制循环同步

## 💡 优化建议

### 降低视频延迟
当前 MJPEG 延迟约 200-500ms，如需更低延迟：
1. 降低摄像头分辨率 (如 640x480)
2. 降低帧率 (如 15 FPS)
3. 升级 5GHz WiFi 网络

### 提高控制精度
调整 `ControlView.tsx` 中的映射参数：
```typescript
const linear = (g29Data.throttle / 100.0) * 2.0;  // 修改 2.0 调整最大速度
const angular = -(g29Data.steering / 45.0) * 1.5; // 修改 1.5 调整转向灵敏度
```

## 📞 常见问题

### Q: 能否同时显示多个摄像头？
A: 可以，在小车 ROS2 中添加多个话题，在 `CarVideoFeed` 组件中创建多个实例。

### Q: 如何保存行车记录？
A: 点击 "结束接管并归档" 会保存当前会话数据。

### Q: 支持键盘控制吗？
A: 当前版本专注 G29，如需键盘控制可在小车网页按 K 切换。
