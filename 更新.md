**原架构 (v1.0)**:
- ❌ 需要 SRS 服务器实现 WebRTC，部署复杂

**现架构 (v2.0)**:
- ✅ **MJPEG 流式传输** - 直接 HTTP 推送，无需额外服务器

- ✅ **一键启动脚本** - 30 秒完成所有服务启动
- ✅ **视频代理架构** - 完美解决跨域问题
- ✅ **Python/Node.js 双版本** - 跨平台支持

## 🏗️ 核心架构变更

### 1️⃣ 服务层增强 (src/services/)

#### 新增：`carControl.ts` - 小车 HTTP 控制服务

**原架构**: 仅通过 WebSocket 接收遥测数据，无控制指令下发通道

**现架构**: 
```typescript
// 新增 HTTP 控制接口
export async function sendCarCommand(type: string, value?: any): Promise<void>
export async function sendVelocityCommand(linear: number, angular: number): Promise<void>
export async function sendStopCommand(): Promise<void>
```

**技术细节**:
- **协议**: HTTP POST (JSON)
- **端点**: `http://{CAR_IP}:8000/control`
- **视频代理**: `http://localhost:8001/` (解决跨域)
- **可配置 IP**: 支持运行时动态配置小车 IP 地址

**使用场景**:
```typescript
// G29 方向盘控制 - 实时转发到小车
sendVelocityCommand(linear, angular)  // 50ms 间隔

// 紧急停止
sendStopCommand()  // 刹车优先
```

---

### 2️⃣ 视频流架构升级

#### 原架构 (v1.0)
```
车端摄像头 → WebRTC (SRS) → 前端 VideoFeed
⚠️ 注意：WebRTC 方案需要 SRS 服务器，部署复杂
```

#### 现架构 (v2.0) - **MJPEG 流式传输 (生产方案)**
```
车端摄像头 → HTTP MJPEG 流 → 视频代理 (8001) → 前端 CarVideoFeed
```

**技术选型说明**:

| 方案 | 延迟 | 部署复杂度 | 兼容性 | 现架构采用 |
|:-----|:----:|:----------:|:------:|:----------:|
| WebRTC | 200-500ms | ⭐⭐⭐⭐⭐ (需 SRS 服务器) | 中 | ❌ 演示用 |
| **MJPEG** | 300-800ms | ⭐ (直接 HTTP) | 高 | ✅ **生产用** |

**现架构数据流**:
```
1. 车端推流
   python3 start_robot.sh
   # FFmpeg 推送 MJPEG 流到 http://cloud-server:8001/

2. 视频代理转发 (解决跨域)
   node video-stream-proxy.cjs
   # 监听 8001 端口，转发车端视频流

3. 前端播放
   <CarVideoFeed src="http://localhost:8001/" />
   # 使用 <img> 标签直接显示 MJPEG 流
```

**关键变更**:
1. **新增 `CarVideoFeed.tsx`**: 专为 MJPEG 流优化，使用 `<img>` 标签直接显示流式图像
   ```tsx
   // 核心实现：MJPEG 流通过 img 标签的 src 自动刷新
   <img 
     src={`${streamUrl}?timestamp=${Date.now()}`}
     className="w-full h-full object-cover"
   />
   ```

2. **保留 `VideoFeed.tsx`**: 保留 WebRTC 支持代码 (演示/未来扩展)

3. **多级视频代理**:
   - `video-stream-proxy.cjs` - 基础代理 (端口 8001) ✅ **生产使用**
   - `video-proxy-lowlatency.cjs` - 低延迟优化 (可选)
   - `video-proxy-ultra.cjs` - 超低延迟模式 (可选)

**配置方式**:
```javascript
// LocalStorage 配置 - MJPEG 方案
localStorage.setItem('car_ip', '192.168.1.100')  // 小车 IP
// 视频流自动通过 http://localhost:8001/ 代理
```

### 3️⃣ G29 方向盘服务升级

#### 原架构
```
G29 硬件 → g29-server.cjs (Node.js) → WebSocket:8080 → ControlView
```

#### 现架构 - 双版本支持
```
方案 A: Node.js 版本
G29 → g29-server.cjs → WebSocket:8080 → ControlView

方案 B: Python 版本
G29 → g29_sender.py → WebSocket:8080 → ControlView
```

**数据格式** (保持不变):
```json
{
  "status": "connected",
  "steering": -450~450,
  "throttle": 0~100,
  "brake": 0~100
}
```

**Python 版本优势**:
- 跨平台支持 (Linux/macOS/Windows)
- 更适合嵌入式设备部署
- 依赖更轻量 (`inputs` 库)

### 4️⃣ 视图层优化 (src/views/)

#### ControlView.tsx 变更
```diff
+ import { sendVelocityCommand, sendStopCommand } from '../services/carControl'
+ import { CarVideoFeed } from '../components/CarVideoFeed'

- 仅展示遥测数据
+ 实时发送控制指令到小车 (50ms 间隔)
```

**核心逻辑**:
```typescript
// G29 数据转发到小车
useEffect(() => {
  if (g29Data.status !== 'connected') {
    sendStopCommand()
    return
  }
  
  const interval = setInterval(() => {
    if (g29Data.brake > 10) {
      sendStopCommand()  // 刹车优先
    } else {
      const linear = (g29Data.throttle / 100.0) * 2.0
      const angular = -(g29Data.steering / 45.0) * 1.5
      sendVelocityCommand(linear, angular)
    }
  }, 50)
}, [g29Data])
```

#### MonitorView.tsx 变更
- 新增 **CarVideoFeed** 组件集成
- 增强型 HUD 显示 (胎压、激光雷达状态)
- 优化状态告警动画 (脉冲效果)

### 5️⃣ 启动流程优化

#### 原架构
```bash
# 需手动启动多个服务
node g29-server.cjs
npm run dev
# 单独配置 SRS Docker
```

#### 现架构 - 一键启动
```bash
# Windows
start-all.bat

# Linux/macOS
./start-all.sh (待创建)
```

**start-all.bat 执行流程**:
```batch
[1/3] 启动视频流代理 (video-stream-proxy.cjs) → 端口 8001
[2/3] 启动 G29 方向盘服务 (g29-server.cjs) → 端口 8080
[3/3] 启动 Web 平台 (npm run dev) → 端口 5173
```

**配套脚本**:
| 脚本 | 功能 |
|:-----|:-----|
| `start-all.bat` | 一键启动所有服务 |
| `start-lowlatency.bat` | 低延迟模式启动 |
| `kill-port.bat` | 释放占用端口 |
| `car-stream.sh` | 车端推流脚本 |
| `start_robot.sh` | 小车控制端启动 |

| **C** | 控制指令下发通道 | **已实现**: `carControl.ts` HTTP 接口 |

**真实部署示例**:
```typescript
// [填空区 A] - 遥测数据连接 (真实车端)
const socket = new WebSocket(`ws://${CAR_IP}:8000/telemetry`)
socket.onmessage = (e) => {
  const data = JSON.parse(e.data)
  this.notifySubscribers(data)
}

// [填空区 C] - 控制指令下发 (已实现)
export async function sendVelocityCommand(linear, angular) {
  return sendCarCommand('set_vel', { linear, angular })
}
```

### 车端配置 (真实部署)

#### 📌 推荐方案：MJPEG 流式传输

```bash
# 车端运行 FFmpeg 推送 MJPEG 流
ffmpeg -f v4l2 -i /dev/video0 \
  -f mjpeg -update 1 \
  http://cloud-server:8001/
```

**车端启动脚本** (`start_robot.sh`):
```bash
#!/bin/bash
# 启动小车控制服务
python3 robot_web_control.py

# 启动摄像头推流 (MJPEG)
ffmpeg -f v4l2 -i /dev/video0 \
  -f mjpeg -q:v 5 \
  http://YOUR_CLOUD_IP:8001/
```

#### 可选方案：WebRTC (演示/未来扩展)

```bash
# 使用 SRS Docker (需要额外配置)
docker run --rm -p 1935:1935 -p 1985:1985 -p 8080:8080 \
  registry.cn-hangzhou.aliyuncs.com/ossrs/srs:4 \
  ./objs/srs -c srs-low-latency.conf
```
