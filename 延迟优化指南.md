## ⚡ 方案 1: 优化 HTTP 代理（推荐，简单有效）

### 使用优化版代理服务器

已创建 `video-proxy-lowlatency.cjs`，关键优化：
- ✅ 禁用 HTTP 代理 (`agent: false`)
- ✅ 立即转发每个 chunk（不缓冲）
- ✅ Keep-Alive 长连接
- ✅ 禁用缓存

### 启动方式

```bash
# 设置小车 IP（可选，默认 192.168.0.5）
export CAR_IP=192.168.0.5

# 启动代理
node video-proxy-lowlatency.cjs
```

### Web 平台配置

在浏览器控制台运行：
```javascript
localStorage.setItem('car_ip', 'localhost');
location.reload();
```

## 🔧 方案 2: SSH 端口转发（无需额外软件）

### 在电脑上运行

```bash
# SSH 端口转发（需要小车支持 SSH）
ssh -L 8001:localhost:8000 nvidia@小车IP

# 或者 root 用户
ssh -L 8001:localhost:8000 root@小车IP
```

### 保持连接（防止断开）

创建 `~/.ssh/config`（Windows: `C:\Users\你的用户名\.ssh\config`）：

```
Host car-robot
    HostName 小车 IP
    User nvidia
    LocalForward 8001 localhost:8000
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

然后运行：
```bash
ssh car-robot
```

### Web 平台配置

```javascript
localStorage.setItem('car_ip', 'localhost');
localStorage.setItem('video_port', '8001');
location.reload();
```

## 🌐 方案 3: 直接访问小车（最低延迟，需同网络）

### 如果电脑和小车在同一局域网

直接访问小车视频流，无需代理：

```
http://小车 IP:8000/mjpeg
```

### 修改前端代码

编辑 `src/services/carControl.ts`（如果存在）或相关视频组件：

```typescript
// 将视频流地址改为小车直连
const VIDEO_URL = `http://${CAR_IP}:8000/mjpeg`;
```

## 🎯 方案 4: WebRTC 超低延迟（最佳但复杂）

### 使用 SRS（无需 Docker 的二进制包）

#### 1. 下载 SRS

```bash
# Linux
wget https://github.com/ossrs/srs/releases/download/v5.0.207/srs-v5.0.207.tar.gz
tar -xzf srs-v5.0.207.tar.gz
cd srs-v5.0.207
```

#### 2. 启动 SRS

```bash
./objs/srs -c conf/rtc.conf
```

#### 3. 小车端推流

```bash
# 安装 FFmpeg
sudo apt install ffmpeg

# 推流到 SRS
ffmpeg -f v4l2 -i /dev/video0 \
  -video_size 1280x720 -framerate 30 \
  -c:v libx264 -preset ultrafast -tune zerolatency \
  -b:v 2000k -g 60 \
  -f flv rtmp://SRS服务器IP/live/car_front
```

#### 4. 前端播放

编辑 `src/services/vehicleConnection.ts`:

```typescript
const SRS_SERVER_IP = "你的 SRS 服务器 IP";
const USE_WEBRTC = true;
```

## 🔍 延迟测试

### 方法 1: 浏览器开发者工具

1. 按 `F12` 打开开发者工具
2. 进入 Network 标签
3. 刷新视频流
4. 查看 `Timing` → `TTFB` 和 `Content Download`

### 方法 2: 命令行测试

```bash
# 测试延迟
curl -o /dev/null -s -w "DNS 解析：%{time_namelookup}s\n连接时间：%{time_connect}s\n首字节：%{time_starttransfer}s\n总时间：%{time_total}s\n" \
  http://localhost:8001/mjpeg
```

### 方法 3: 实际延迟测试

用手机秒表对着电脑屏幕，对比实际时间和视频显示时间。

## ⚙️ 性能调优

### 1. 降低分辨率（小车端）

编辑小车摄像头启动参数：

```bash
# 从 1080p 降到 720p
1920x1080 → 1280x720

# 或降到 480p（延迟最低）
854x480
```

### 2. 降低帧率

```bash
# 30fps → 20fps（节省带宽）
framerate=20
```

### 3. 降低码率

```bash
# MJPEG 质量参数
quality=80  # 默认 85
```

### 4. 网络优化

```bash
# 检查网络质量
ping 小车IP

# 查看带宽占用
iftop  # Linux
```

## 🔧 故障排查

```bash
# 1. 检查网络延迟
ping 小车IP

# 2. 检查带宽
# 视频流占用带宽 = 分辨率 × 帧率 × 压缩比
# 720p@30fps ≈ 2-3Mbps

# 3. 降低分辨率
# 编辑小车摄像头配置
```

### 问题 2: 画面卡顿

```bash
# 1. 检查 CPU 使用率
top  # Linux
Task Manager  # Windows

# 2. 检查网络丢包
ping -c 100 小车IP

# 3. 降低码率
# quality=75
```

### 问题 3: 连接断开

```bash
# 1. 检查防火墙
sudo ufw allow 8000/tcp
sudo ufw allow 8001/tcp

# 2. 检查端口占用
netstat -tulpn | grep :8000
```
