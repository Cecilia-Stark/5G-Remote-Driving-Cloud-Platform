## 启动步骤

### 1. 启动小车服务 (SSH 连接到 192.168.0.5)

```bash
# SSH 登录小车
ssh user@192.168.0.5

# 启动 ROS2 视频流服务 (端口 8080)
ros2 run web_video_server web_video_server

# 启动小车控制服务 (端口 8000)
cd ~/robot_control
python3 robot_control.py
```

### 2. 启动本地服务 (Windows)

打开 **3 个命令行窗口**，分别执行以下命令：

#### 窗口 1: 视频流代理服务器

```bash
cd E:\OneDrive\桌面\tt\5G-Remote-Driving-Cloud-Platform
node video-stream-proxy.cjs
```

**功能：** 将小车 8080 端口的视频流转发到本地 8001 端口，解决跨域问题

**输出示例：**
```
============================================================
📺 视频流代理服务器
============================================================
目标地址：http://192.168.0.5:8080
本地端口：http://localhost:8001
============================================================

✅ 代理服务器已启动

视频流地址：
  本地访问：http://localhost:8001/
  远程访问：http://你的电脑 IP:8001/

在 Web 平台配置小车 IP 为：localhost
```

#### 窗口 2: G29 方向盘服务端

```bash
cd E:\OneDrive\桌面\tt\5G-Remote-Driving-Cloud-Platform
node g29-server.cjs
```

**功能：** 接收 G29 方向盘输入，通过 WebSocket 转发到 Web 平台

**输出示例：**
```
============================================================
🎮 G29 方向盘服务器
============================================================
WebSocket 地址：ws://localhost:8080
控制服务地址：http://192.168.0.5:8000/control
============================================================

✅ G29 服务器已启动
等待 G29 设备连接...
```

#### 窗口 3: Web 平台

```bash
cd E:\OneDrive\桌面\tt\5G-Remote-Driving-Cloud-Platform
npm run dev
```

**功能：** React 前端 Web 平台

**输出示例：**
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://你的电脑 IP:5173/
```

### 3. 访问 Web 平台

打开浏览器访问：**http://localhost:5173/**

#### 首次使用

1. **注册账号** - 输入用户名和密码进行注册
2. **配置小车 IP** - 在设置中配置小车 IP 地址（默认：`localhost`）
3. **登录系统** - 使用注册的账号登录

#### 功能页面

- **实时监控** - 查看摄像头视频、车辆状态、地图位置
- **远程控车** - 使用 G29 方向盘控制小车
- **数据回放** - 查看历史行驶数据

| Web 平台 | 5173 | React 前端应用 |

## 常见问题

### 视频流不显示

1. **检查代理服务器是否启动**
   ```bash
   node video-stream-proxy.cjs
   ```

2. **测试视频流连接**
   ```bash
   curl http://localhost:8001/
   ```

3. **检查小车视频服务**
   ```bash
   curl http://192.168.0.5:8080/stream?topic=/image_raw
   ```

4. **刷新浏览器** - 按 `Ctrl+Shift+R` 强制刷新

### G29 方向盘无响应

1. **检查 G29 服务是否启动**
   ```bash
   node g29-server.cjs
   ```

2. **检查 G29 设备连接**
   - 确保 G29 已连接到电脑 USB
   - 确保 G29 驱动已安装

3. **查看浏览器控制台** - 按 `F12` 查看 WebSocket 连接状态

### 小车控制无响应

1. **检查网络连接**
   ```bash
   ping 192.168.0.5
   ```

2. **检查小车控制服务**
   - SSH 登录小车
   - 确认 `robot_control.py` 正在运行

3. **检查跨域配置**
   - 小车 HTTP 服务需要允许跨域访问

## 关闭服务

### 停止本地服务

在对应的命令行窗口按 `Ctrl+C` 停止服务

或使用任务管理器结束以下进程：
- `node.exe` (video-stream-proxy.cjs)
- `node.exe` (g29-server.cjs)
- `node.exe` (vite/dev server)

### 停止小车服务

```bash
# SSH 登录小车后按 Ctrl+C 停止服务
# 或关闭小车电源
```

## 技术栈

- **前端**: React 18 + TypeScript + Vite + TailwindCSS
- **后端**: Node.js (Express/原生 HTTP)
- **通信**: WebSocket (G29), HTTP/MJPEG (视频流)
- **硬件**: Logitech G29 方向盘，USB 摄像头
- **小车**: ROS2, web_video_server, Python HTTP 服务
